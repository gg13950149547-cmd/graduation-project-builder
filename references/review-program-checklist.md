# Program Delivery Review Checklist

Use this checklist before claiming a graduation-project program is complete.

## Required Checks
- [ ] task mode has been classified correctly
- [ ] requirement summary exists
- [ ] backlog exists for missing work
- [ ] startup path is real
- [ ] persistence exists when business data is real
- [ ] auth exists when protected roles or admin flows exist
- [ ] admin side exists when the topic implies management
- [ ] analytics exist when operational/business data exists
- [ ] admin analytics include at least one line chart unless user explicitly overrides
- [ ] demo data path is real
- [ ] business-page copy leads with user intent instead of platform introduction
- [ ] shared shell, hero text, helper text, and empty states have passed a user-perspective copy review
- [ ] package contents match the required bundle for the project class
- [ ] verification results are recorded as passed / skipped / failed

## Report Format
For each unchecked item, explain:
- why it matters
- what evidence is missing
- what should be built or verified next
