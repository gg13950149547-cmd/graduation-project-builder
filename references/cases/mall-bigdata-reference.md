# Mall Big Data Reference

Use this as a compact example of a defendable data-analysis graduation project with a lighter implementation behind a heavier architecture story.

## What Makes It Useful

- keeps the full architecture narrative
- simplifies expensive infrastructure into a runnable local path
- still exposes dashboard, pipeline, export, and sync surfaces
- preserves a defendable demo loop

## Transferable Lessons

- Keep the architecture story, but make the runnable demo path realistic.
- Replace heavy dependencies with explainable simulations when full infrastructure is impractical.
- Preserve pipeline, visualization, and export surfaces so the system still feels complete.

## What Not To Generalize Blindly

- project-specific platform names
- project-specific data sources
- project-specific Hadoop or database sync details

Treat it as a strategy example for constrained implementations, not a direct blueprint.
